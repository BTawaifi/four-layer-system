# Flow 14: Performance Audit

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-set-ground-truth`
   *Description:* Establishes absolute source of truth files that take precedence over other information in the session
2. **Specialist:** `@specialist-technical-writer`
   *Description:* Technical writing expertise focusing on clear documentation, API references, and user guides
3. **Modifier:** `@modifier-conservative-mode`
   *Description:* Prioritizes proven, stable patterns and technologies over experimental approaches, emphasizing reliability and maintainability
4. **Workflow:** `@workflow-performance-audit`
   *Description:* Composite workflow combining performance analysis, optimization focus, and comprehensive system auditing

## Prompt Command
```
@context-set-ground-truth @specialist-technical-writer @modifier-conservative-mode @workflow-performance-audit
```

## Example Usage Prompt
> Please apply the `@context-set-ground-truth` rules to establish the working environment, act as a `@specialist-technical-writer`, keep the responses `@modifier-conservative-mode`, and execute the `@workflow-performance-audit` on the current module.
