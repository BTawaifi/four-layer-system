# Flow 48: Create New Rule

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-state-define-task-list`
   *Description:* Define and track a task list for multi-step processes with progress management
2. **Specialist:** `@specialist-javascript-developer`
   *Description:* JavaScript development expertise covering frontend, backend, and full-stack applications with modern frameworks and best practices
3. **Modifier:** `@modifier-target-language-version`
   *Description:* Ensures generated code is compatible with specific language versions, avoiding newer features
4. **Workflow:** `@workflow-create-new-workflow-rule`
   *Description:* Creates new workflow rules for automating repetitive tasks with interactive prompts and sequential execution

## Prompt Command
```
@context-state-define-task-list @specialist-javascript-developer @modifier-target-language-version @workflow-create-new-workflow-rule
```

## Example Usage Prompt
> Please apply the `@context-state-define-task-list` rules to establish the working environment, act as a `@specialist-javascript-developer`, keep the responses `@modifier-target-language-version`, and execute the `@workflow-create-new-workflow-rule` on the current module.
