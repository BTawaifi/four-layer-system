# Flow 38: Accessibility Planning

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-set-ground-truth`
   *Description:* Establishes absolute source of truth files that take precedence over other information in the session
2. **Specialist:** `@specialist-javascript-developer`
   *Description:* JavaScript development expertise covering frontend, backend, and full-stack applications with modern frameworks and best practices
3. **Modifier:** `@modifier-think-step-by-step`
   *Description:* Requires explicit step-by-step thinking process to be shown before providing final answers
4. **Workflow:** `@workflow-accessibility-planning`
   *Description:* Accessibility audit planning and scope definition for WCAG compliance assessment

## Prompt Command
```
@context-set-ground-truth @specialist-javascript-developer @modifier-think-step-by-step @workflow-accessibility-planning
```

## Example Usage Prompt
> Please apply the `@context-set-ground-truth` rules to establish the working environment, act as a `@specialist-javascript-developer`, keep the responses `@modifier-think-step-by-step`, and execute the `@workflow-accessibility-planning` on the current module.
