# Flow 27: Generate Tests

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-establish-aliases`
   *Description:* Establishes contextual aliases for easier reference to different parts of the codebase during development sessions
2. **Specialist:** `@specialist-product-manager`
   *Description:* Product management expertise focusing on user needs, business value, and feature prioritization
3. **Modifier:** `@modifier-react-best-practices`
   *Description:* Enforces React-specific best practices, patterns, and performance optimizations
4. **Workflow:** `@workflow-generate-tests`
   *Description:* Generates comprehensive test suites with unit tests, edge cases, error handling, and UI testing

## Prompt Command
```
@context-establish-aliases @specialist-product-manager @modifier-react-best-practices @workflow-generate-tests
```

## Example Usage Prompt
> Please apply the `@context-establish-aliases` rules to establish the working environment, act as a `@specialist-product-manager`, keep the responses `@modifier-react-best-practices`, and execute the `@workflow-generate-tests` on the current module.
