# Flow 35: Refactor Generalize

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-set-ground-truth`
   *Description:* Establishes absolute source of truth files that take precedence over other information in the session
2. **Specialist:** `@specialist-security-auditor`
   *Description:* White hat hacker perspective specializing in vulnerability identification and secure coding practices
3. **Modifier:** `@modifier-cloud-agnostic`
   *Description:* Design solutions that work across multiple cloud providers with minimal vendor lock-in
4. **Workflow:** `@workflow-refactor-generalize`
   *Description:* Generalizes successful fixes by finding and applying similar patterns consistently across the codebase

## Prompt Command
```
@context-set-ground-truth @specialist-security-auditor @modifier-cloud-agnostic @workflow-refactor-generalize
```

## Example Usage Prompt
> Please apply the `@context-set-ground-truth` rules to establish the working environment, act as a `@specialist-security-auditor`, keep the responses `@modifier-cloud-agnostic`, and execute the `@workflow-refactor-generalize` on the current module.
