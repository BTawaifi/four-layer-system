# Flow 30: Progressive System Architecture Docs

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-progressive-context-loader`
   *Description:* Context for intelligent loading and management of large information contexts across Cursor IDE sessions
2. **Specialist:** `@specialist-ux-researcher`
   *Description:* User experience research expertise focusing on user behavior analysis, usability testing, and human-centered design
3. **Modifier:** `@modifier-no-new-dependencies`
   *Description:* Ensures solutions use only existing dependencies and standard library, prohibiting introduction of new third-party packages
4. **Workflow:** `@workflow-progressive-system-architecture-docs`
   *Description:* Progressive system architecture documentation workflow that builds comprehensive architectural documentation across multiple focused sessions

## Prompt Command
```
@context-progressive-context-loader @specialist-ux-researcher @modifier-no-new-dependencies @workflow-progressive-system-architecture-docs
```

## Example Usage Prompt
> Please apply the `@context-progressive-context-loader` rules to establish the working environment, act as a `@specialist-ux-researcher`, keep the responses `@modifier-no-new-dependencies`, and execute the `@workflow-progressive-system-architecture-docs` on the current module.
