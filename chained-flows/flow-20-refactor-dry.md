# Flow 20: Refactor Dry

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-state-set-primary-goal`
   *Description:* Establishes the primary goal for the entire session and ensures all suggestions align with achieving that objective
2. **Specialist:** `@specialist-data-scientist`
   *Description:* Data science expertise focusing on statistical analysis, exploratory data analysis, modeling, and data-driven insights
3. **Modifier:** `@modifier-mobile-first`
   *Description:* Prioritizes mobile-first design principles, responsive layouts, and mobile user experience optimization
4. **Workflow:** `@workflow-refactor-dry`
   *Description:* Refactors code to eliminate duplication by applying the DRY principle with data structures and dynamic approaches

## Prompt Command
```
@context-state-set-primary-goal @specialist-data-scientist @modifier-mobile-first @workflow-refactor-dry
```

> Please apply the `@context-state-set-primary-goal` rules to establish the working environment, act as a `@specialist-data-scientist`, keep the responses `@modifier-mobile-first`, and execute the `@workflow-refactor-dry` on the current module.
