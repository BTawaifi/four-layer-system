# Flow 4: Create New Context Rule

**Scenario:** Integrating a full chained flow combining domain expertise, operational constraints, state management, and the execution workflow.

## Components Used
1. **Context:** `@context-token-efficiency-manager`
   *Description:* Context for monitoring and optimizing token usage in complex Cursor IDE workflows
2. **Specialist:** `@specialist-frontend-developer`
   *Description:* Frontend development expertise specializing in React, Vue, Angular, and modern web development practices
3. **Modifier:** `@modifier-code-only`
   *Description:* Ensures responses contain only raw code blocks without explanations, introductions, or markdown formatting
4. **Workflow:** `@workflow-create-new-context-rule`
   *Description:* Factory workflow for creating new context management rules with interactive prompts and proper structure

## Prompt Command
```
@context-token-efficiency-manager @specialist-frontend-developer @modifier-code-only @workflow-create-new-context-rule
```

## Example Usage Prompt
> Please apply the `@context-token-efficiency-manager` rules to establish the working environment, act as a `@specialist-frontend-developer`, keep the responses `@modifier-code-only`, and execute the `@workflow-create-new-context-rule` on the current module.
